# 算命网页设计参考

**这是我对"算命网页该怎么做"的初步想法，供主人拍板参考。**

---

## 一、产品定位选择

### 定位 A：文化科普型
- 主打"了解中国传统术数文化"
- 每种方法都简单介绍 + 提供一个"体验入口"
- **合规风险最低**，可以走完整商业化
- 适合：文创、国风品牌联动、内容社区

### 定位 B：娱乐/趣味型
- 主打"每日一卦""今日运势""AI 面相"
- 强互动，社交属性（分享结果到朋友圈）
- **合规相对安全**（免费或象征性收费，明确娱乐用途）
- 适合：小程序、H5 传播

### 定位 C：工具型
- 主打"专业排盘工具"给已入门的爱好者
- 只提供数据不提供解读，或提供教科书式解读
- **合规安全**（工具属性）
- 适合：垂直用户，付费空间小但精准

### 定位 D：付费算命服务
- 用户付费获取"个性化命理解读"
- ⚠️ **合规风险最高**（"利用封建迷信虚构事实"入刑风险）
- 尤其如果附带"改运物品/开运仪式"，几乎必被定为诈骗
- **强烈不建议这个方向**

**建议：A + B 组合，工具体验 + 内容科普。**

## 二、技术方案建议

### 前端

- **框架**：React / Next.js（如果要 SEO 好）或 Vue
- **UI 风格建议**：
  - 底色：深色调（黑、深蓝、深红），配金色描线
  - 元素：太极、八卦符号、云纹、篆书字体
  - 动效：起卦时铜钱下落动画、罗盘旋转动画
  - 参考："橙子相""易奇八字"等成熟产品的视觉

### 核心算法库（开源可直接用）

| 需求 | 库 |
|---|---|
| 农历/公历转换、干支、节气 | `lunar-javascript`（npm，成熟） |
| 紫微斗数排盘 | `iztro`（npm，中文文档齐全） |
| 八字排盘 | `lunar-javascript` 已带四柱 |
| 六爻/梅花易数起卦 | 自己写，逻辑简单 |
| 周易 64 卦文本 | 网上公版数据（无版权） |

### 后端

- **无用户数据版**：纯静态站点（Cloudflare Pages / Vercel），所有算命在前端算
- **需要保存历史**：Node.js + SQLite / Turso（主人已有 Turso 账号，见 MEMORY）

### AI 加持（可选亮点）

- **接 LLM 生成解读文案**：用户排盘完，把命盘数据给 LLM，生成个性化解读段落
- **成本敏感**：可以设置每日免费次数 + 付费购买更多次数
- **注意**：LLM 生成的内容明确标注"娱乐参考"

## 三、功能模块建议（MVP → 完整版）

### MVP（最小可用版本）

1. **八字排盘 + 基础解读**
   - 输入生辰 → 出 8 个字 + 五行分布 + 十神
   - 免费
2. **周易随机抽卦**
   - "今日一卦"点一下就出
   - 免费
3. **姓名笔画数理**
   - 输入名字 → 出五格数理
   - 免费

### 完整版（V1.0）

4. **紫微斗数命盘**
   - 可视化 12 宫命盘（漂亮，是主打卖点）
   - 免费或象征性收费
5. **六爻起卦**
   - 摇卦动画 → 得卦 → 白话解读
   - 免费
6. **梅花易数**
   - 输入数字/时间 → 出卦
   - 免费
7. **奇门遁甲盘**
   - 时家奇门九宫盘可视化
   - 高级功能
8. **黄历 / 每日宜忌**
   - 首页常驻
   - 免费

### 拓展（V2.0）

9. **AI 面相**（拍照识别 + LLM 解读）
10. **AI 手相**（同上）
11. **LLM 深度解读**（付费点）
12. **社交分享**（生成命盘海报）

## 五、合规风险要点（一定要看）

摘自维基百科"算命"条目：

> 現行中華人民共和國刑法中算命收費行為並未歸為明定違法，但實務上有相當風險被定為詐騙罪。其界定標準線在於：
> 1. **創造的利潤金額是否較高**
> 2. **是否有人報警**
> 3. 偵查階段後如果無法舉證本身有命理學相關研究知識、一切算命內容有學理依據
> 4. 同時有所謂**交更多錢能改運的服務**（例如買開運小物或開運儀式）
> 
> **符合以上狀況時有很高機率被以「使用封建迷信虛構事實詐騙」而定罪。**

### 我的建议红线

- ✅ 免费的算命工具 + 内容科普
- ✅ 象征性收费（比如会员制解锁高级功能，不主打"算命"本身）
- ✅ 页面明确标注"仅供娱乐/文化参考"
- ❌ 不要卖"开运物"（水晶、貔貅、护符）
- ❌ 不要有"付费改运/化解"服务
- ❌ 不要承诺预测准确度、不要说"大师亲算"

## 六、下一步问题（等主人拍板）

以下问题我需要主人回答，才能进入具体设计：

1. **定位选哪个？**（A 科普 / B 娱乐 / C 工具 / A+B）
2. **MVP 优先做哪几个？**（我建议：八字 + 紫微 + 每日一卦）
3. **是否要 AI 生成解读？**（涉及 LLM 成本和用户体验）
4. **要不要用户系统？**（保存历史命盘、多命盘管理）
5. **技术栈偏好？**（React/Vue/纯静态？部署到 Cloudflare Pages 还是自建？）
6. **视觉风格偏好？**（传统道家风 / 现代简约国风 / 二次元趣味风）
7. **目标用户？**（年轻人娱乐 / 中年人严肃求测 / 命理爱好者）
8. **是否要移动端 App？**（小程序 / H5 / 原生）

**给我几个方向定下来，我就可以开始画原型（HTML 交互稿）+ 写 PRD。**
