## 各方法的开发复杂度

| 方法 | 计算难度 | 数据量 | 网页可行性 | 建议 |
|------|----------|--------|-----------|------|
| 八字 | ★★★★ | 大（万年历+节气+经纬度）| ★★★★★ | 必做，MVP第一版 |
| 塔罗 | ★ | 中（78张牌意）| ★★★★★ | 必做，最容易实现 |
| 梅花易数 | ★★ | 中（64卦爻辞）| ★★★★★ | 推荐，交互简单 |
| 六爻 | ★★★★★ | 大（纳甲6亲世应用神）| ★★★★ | 进阶，需完整64卦数据 |

## MVP 建议开发顺序

**第一版（1-2周）**：
1. 塔罗（每日一抽 + 三张牌）→ 上手最快，用户体验好
2. 梅花易数（时间起卦 + 数字起卦）→ 免排盘，起卦即断

**第二版（2-4周）**：
3. 八字（简易排盘 + 基础解读）→ 需接入万年历/节气 API
4. 六爻（摇卦 + 世应六亲）→ 需完整 64 卦数据库

## 关键技术依赖

**八字排盘需要**：
- 万年历（阳历→阴历+干支）
- 24 节气日期时间表
- 城市经纬度库
- 真太阳时修正算法

**推荐开源库**：
- `@thewei/lunar` (npm) - 农历+八字工具
- `chinese-lunar-calendar` - 农历转换
- `sxtwl` - 万年历（含节气）

**塔罗需要**：
- 78 张牌图片（有免费公版素材）
- 洗牌 + 抽牌算法

## 数据来源

- 天干地支、五行、十神、六冲六合：传统命理典籍常识（几百年公开知识）
- 八卦、64卦、纳甲、六亲、世应：《易经》、《周易本义》等
- 塔罗78张牌意：Mark McElroy《A Guide to Tarot Meanings》(公版素材)
- 维基百科相关条目（已在 raw/ 和 text/ 目录）

## 合规提示

**参考"03-web-design-notes.md"中的合规风险部分：**
- ⚠️ 中华人民共和国刑法背景下，商业化"付费改运"、"开运物"有诈骗定罪风险
- ✅ 定位为**娱乐/文化科普**较为安全
- ✅ 界面显眼位置声明"仅供娱乐参考"
- ❌ 不承诺具体人事的确定性预测
- ❌ 不销售"开运物品"
