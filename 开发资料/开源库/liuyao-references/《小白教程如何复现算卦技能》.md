## 一、什么是 Skill？30秒理解

Skill = 你给 Claude 写的一套**操作手册**。

但重点来了——**这个手册你也不用自己写，你告诉 Claude 你要什么手册，它自己写。**

| 你要做的 | Claude 做的 |
|---|---|
| 描述你想要什么功能 | 规划文件结构 |
| 回答 Claude 的确认问题 | 编写 SKILL.md（主流程文件） |
| 审核成品，提修改意见 | 编写所有参考资料文件 |
| 测试，说哪里不对 | 修改调整 |

**你全程不需要手写任何 Markdown 文件。**

## 二、前置准备

### 安装 Claude Code

```bash
npm install -g @anthropic-ai/claude-code
```

需要：
- Node.js 18+
- Anthropic API key 或 Claude Max/Pro 订阅

验证安装：
```bash
claude --version
```

### 你需要准备的"原材料"

Claude 很聪明，但它需要你提供**领域知识的方向和要求**。以算卦为例，你心里要清楚：

- 你想做什么类型的占卜（六爻？梅花？奇门？）
- 基于什么理论体系（哪本书？哪个流派？）
- 需要哪些功能（起卦、装卦、断卦？）
- 输出格式的期望（要画卦象表吗？要标注世应吗？）

**不需要你写出来，能说清楚就行。** Claude 会追问细节。

## 三、真实工作流：三步搞定

```
┌─────────────────────────────────────────────┐
│  第一步：建项目文件夹，启动 Claude Code       │
│         （1分钟）                             │
├─────────────────────────────────────────────┤
│  第二步：输入 /plan，和 Claude 聊清楚需求     │
│         - Claude 提问，你回答                 │
│         - 来回几轮，确认方案                  │
│         （10-30分钟，取决于需求复杂度）        │
├─────────────────────────────────────────────┤
│  第三步：确认方案后，Claude 自动创建所有文件   │
│         - SKILL.md                           │

│         （反复调试直到满意）                   │

```

就这么简单。下面详细展开每一步。

## 四、第一步：建文件夹 + 启动 Claude Code

打开终端：

```bash
# 创建项目目录
mkdir ~/Desktop/算命
cd ~/Desktop/算命

# 启动 Claude Code
claude
```

这就够了。连 `.claude/skills/` 目录都不需要你手动建，Claude 会自己建。

## 五、第二步：用 /plan 和 Claude 聊清楚需求

### 什么是 /plan？

`/plan` 是 Claude Code 的**规划模式**。进入这个模式后：
- Claude **只规划、不动手**（不会创建或修改任何文件）
- 你和 Claude 反复讨论，直到方案满意
- 确认后 Claude 才开始执行

### 怎么操作

在 Claude Code 对话中输入：

```
/plan
```

然后用自然语言描述你的需求。以下是算卦技能的**真实对话示例**：

---

### 对话示例（你可以直接参考）

**你**：
```
我想做一个六爻占卜的 skill，基于朱熹《周易本义》的卦辞爻辞，
断卦体系用野鹤老人《增删卜易》。

功能要完整：从起卦、装卦到断卦全流程。
包括纳甲、定世应、配六亲、配六神、分析用神旺衰、月建日辰影响、
动爻分析、特殊情况（空亡、入墓、反伏吟等）、应期推断。

要能画出标准的卦象表，标注世应、动爻、变爻。
```

**Claude 可能会问**：
```
好的，几个问题确认一下：

1. 起卦方式：用户直接输入六爻数字（6/7/8/9），还是需要模拟摇卦？
2. 日期处理：需要自动将公历转干支吗？用户不提供日期时用当天？
3. 卦辞来源：64卦卦辞需要我整理成参考文件，爻辞是否也需要？
   还是可以用我训练数据中的知识？
4. 输出详细程度：断卦分析要多详细？简要结论还是完整推理过程？
```

**你**：
```
1. 用户输入6/7/8/9数字
2. 是，自动转干支，默认当天
3. 卦辞整理成文件，爻辞用你自带的知识就行
4. 要完整推理过程，先展示卦象表，然后分析，最后给结论和应期
```

**Claude 会输出一个完整的计划**，包括：
- 文件结构
- 每个文件要写什么内容
- SKILL.md 的大纲
- references/ 每个文件的用途

**你确认后**，Claude 退出规划模式，开始执行。

---

### /plan 过程中的沟通要点

| 你要说清楚的 | 示例 |
|---|---|
| **做什么** | "六爻占卜，完整流程" |
| **基于什么** | "理论用《增删卜易》，卦辞用《周易本义》" |
| **输入是什么** | "用户给6个数字(6/7/8/9)和占问事项" |
| **输出长什么样** | "要画卦象表，要有完整分析过程" |
| **特殊要求** | "大文件用Grep搜索别全读"，"公历要能转干支" |

**不需要说清楚的**（Claude 会自己决定）：
- 文件具体怎么组织
- Markdown 表格怎么画
- 每个文件多少行
- 代码/格式细节

## 六、第三步：让 Claude 自己写所有文件

确认计划后，Claude 会自动：

1. 创建 `.claude/skills/算卦/` 目录
2. 写 `SKILL.md` — 完整的操作流程文件（约 200 行）
3. 创建 `references/` 目录
4. 逐个写参考文件：
   - `najia-table.md` — 八卦纳甲表（141行）
   - `shiyao-table.md` — 64卦世应表（120行）
   - `liuqin-rules.md` — 六亲规则（68行）
   - `liushen-rules.md` — 六神规则（59行）
   - `gua-ci.md` — 64卦卦辞（643行）
   - `yao-ci.md` — 爻辞索引（106行）
   - `duangua-core.md` — 核心断卦规则（224行）
   - `duangua-advanced.md` — 进阶断卦规则（309行）

**整个过程你只需要看着 Claude 创建文件，偶尔确认权限（允许写文件）。**

Claude 会一边写一边解释自己在做什么，你可以随时说"这个不对"或"这里要改"。

## 七、验收和调试

### 第一轮测试

文件创建完毕后，**重新启动 Claude Code**（让它重新加载技能）：

```bash
# 退出当前会话
# 按 Ctrl+C 或输入 /exit

# 重新进入
cd ~/Desktop/算命
claude
```

然后测试：

```
帮我算一卦
```

或直接：

```
/算卦
```

### 发现问题怎么办

直接告诉 Claude 哪里不对就行：

```
卦象表的格式不对，世应标记位置错了，帮我改一下 SKILL.md 里的格式模板
```

```
断卦分析里月建旺衰判断有误，寅月木应该旺不是相，检查一下 duangua-core.md
```

```
六十四卦世应表里"水地比"的世应位置写错了，应该是世三应六
```

Claude 会直接修改对应的文件。改完后再测试，循环直到满意。

### 验收清单

- [ ] 输入引导：Claude 是否正确引导用户输入占问事项和六爻数据？
- [ ] 装卦正确：纳甲、世应、六亲、六神是否配置正确？
- [ ] 卦象表：格式是否整齐，标注是否正确？
- [ ] 断卦分析：月建日辰影响是否判断正确？
- [ ] 卦辞爻辞：引用是否准确？
- [ ] 应期推断：是否有给出？逻辑是否通顺？

## 八、最终文件结构（Claude 生成的成品）

以下是 Claude 写完所有文件后的目录结构：

```
算命/
└── .claude/
    ├── settings.local.json          ← 权限配置（可选）
    └── skills/
        └── 算卦/
            ├── SKILL.md             ← ★ 核心：完整操作流程（223行）
            └── references/
                ├── najia-table.md   ← 纳甲天干地支表（141行）
                ├── shiyao-table.md  ← 64卦八宫世应表（120行）
                ├── liuqin-rules.md  ← 六亲规则+用神表（68行）
                ├── liushen-rules.md ← 六神规则+含义（59行）
                ├── gua-ci.md       ← 64卦卦辞原文（643行）
                ├── yao-ci.md       ← 爻辞阴阳索引（106行）
                ├── duangua-core.md  ← 核心断卦规则（224行）
                └── duangua-advanced.md ← 进阶断卦规则（309行）
```

### 各文件的角色

```
SKILL.md（导演）
    │
    ├──"第1步纳甲" ──→ 读 najia-table.md
    ├──"第2步定世应" ──→ 读 shiyao-table.md
    ├──"第3步配六亲" ──→ 读 liuqin-rules.md
    ├──"第4步配六神" ──→ 读 liushen-rules.md
    ├──"查卦辞" ──→ Grep搜索 gua-ci.md（太大不全读）
    ├──"查爻辞" ──→ 参考 yao-ci.md + Claude自带知识
    ├──"基础断卦" ──→ 读 duangua-core.md
    └──"进阶断卦" ──→ 读 duangua-advanced.md
```

SKILL.md 就像导演，告诉 Claude 每一步该做什么、该去查哪个文件。
references/ 里的文件就是演员手里的台本和道具。

## 九、和 Claude 聊需求的技巧

这是整个教程最实用的部分——**怎么跟 Claude 聊出好的 Skill**。

### 技巧1：先说目标，别说实现

```
✅ 好："我要一个六爻占卜系统，能从用户给的数字推算出完整卦象并断吉凶"
❌ 差："帮我建一个 .claude/skills/算卦 目录，里面放 SKILL.md..."
```

让 Claude 决定怎么组织文件，你只管说要什么结果。

### 技巧2：给 Claude 你的"教材"

如果你有领域知识的参考来源，直接告诉 Claude：

```
"断卦理论用《增删卜易》的体系"
"卦辞用朱熹《周易本义》原文"
"纳甲用京房易学的方法"
```

Claude 的训练数据里有这些书的内容，它能提取出正确的知识来写参考文件。

### 技巧3：对输出格式要具体

```
✅ 好："卦象表要用 ASCII 画，上面标六神六亲天干地支，动爻标○和×，世应标在爻后面"
❌ 差："展示结果要好看"
```

### 技巧4：用 /plan 反复讨论，别急着动手

/plan 模式下可以不断修改方案，不会创建任何文件。一旦退出 /plan 开始执行，改起来就比较麻烦。所以：

```
规划阶段多花时间 → 执行阶段少返工
```

### 技巧5：分批验收

如果技能很复杂（比如算卦有 8 个参考文件），可以让 Claude 先写一部分，你验证没问题后再继续：

```
"先写 SKILL.md 和纳甲表，我验证一下纳甲是否正确"
"纳甲没问题，继续写世应表和六亲规则"
```

### 技巧6：测试时用已知答案来验证

用你已经知道答案的案例来测试：

```
"帮我排一下乾为天卦，六爻全是7，我验证一下装卦对不对"
"用水地比、九五爻动来测试，我知道正确答案是什么"
```

## 十、常见问题

### Q1：/plan 具体怎么用？

在 Claude Code 对话中直接输入 `/plan`，Claude 会进入规划模式。在这个模式下你可以自由讨论方案。讨论完毕后 Claude 会把计划写成文件让你确认，你同意后它就开始执行。

### Q2：Claude 写的文件有错怎么办？

直接说哪里错了就行。比如：
- "世应表里天地否应该是世三应六，你写成世二应五了"
- "坎卦的纳甲应该是戊寅、戊辰、戊午，不是戊子开头"

Claude 会直接修改对应文件。

### Q3：能不能在已有的 Skill 基础上修改？

可以。进入项目目录启动 Claude Code，直接说：
```
"帮我在算卦技能里加一个梅花易数的起卦方式"
"把断卦部分加上六神分析的详细规则"
```

Claude 会读取现有文件，在此基础上修改。

### Q4：可以有多个技能吗？

可以。每个技能一个文件夹：

```
.claude/skills/
├── 算卦/
│   └── SKILL.md + references/
├── 塔罗/
│   └── SKILL.md + references/
└── 做菜/
    └── SKILL.md + references/
```

每个技能独立触发，互不干扰。

### Q5：别人怎么用我的技能？

把项目文件夹（含 `.claude` 目录）打包分享即可。对方解压后：
```bash
cd 算命/
claude
```
输入触发词就能用。也可以放 GitHub 仓库分享。

### Q6：我完全不懂六爻，能做其他技能吗？

当然。你只需要懂你自己的领域就行。同样的流程：
1. `cd 你的项目目录 && claude`
2. `/plan`
3. 用自然语言描述你要什么
4. Claude 规划 → 你确认 → Claude 写文件
5. 测试调试

## 十一、举一反三

同样的 `/plan → 聊需求 → Claude 写文件` 流程，可以做任何领域的技能：

```
├── SKILL.md         → 抽牌流程、牌阵规则、解读流程

    ├── minor-arcana.md  → 56张小牌含义

```

```
你跟 Claude 说：
"做一个中医九种体质辨识 skill，通过问答判断体质类型，
 给出饮食、运动、穴位调理建议。基于王琦教授的体质学说。"

Claude 会自动创建：
├── SKILL.md            → 问诊流程、评分规则、输出格式
```

```
你跟 Claude 说：
"做一个代码审查 skill，按我们团队的规范检查代码，

├── SKILL.md           → 审查流程、检查优先级、报告格式
└── references/
```
